## When to use

The main use case for utility classes is for building layouts around existing components, like buttons and form fields.
Using utility classes means you can avoid naming a CSS class and creating a CSS file for things like wrappers/containers and other
one-off elements. The values for the classes are derived from the `$bdl-grid-unit` variable to make it easy to keep in sync with design
expectations.

Example:

<div className="bdl-u-p-4" style={{ background: variables.bdlBoxBlue20 }}>
    <Ghost style={{float: 'right'}} width="1em" height="1em" />
    <Ghost width={450} className="bdl-u-mr-2"/> <Ghost width="3em" height="1em" /><br />
    <Ghost width={450} className="bdl-u-mt-1"/>
    <div>
        <Ghost width={200} className="bdl-u-mv-8" style={{ height: variables.gridUnit }}/>
    </div>
    <Ghost width={50} height={50}  className="" />
    <span className="bdl-u-mt-1 bdl-u-ml-4">
        <Ghost width={250} height={50} className="" />
    </span>
</div>

## Conventions

The convention is `.bdl-u-pt-0` (prefix-class-modifier-scale)

-   `.bdl-u-` SUIT CSS prefix
-   `m` margin or `p` padding
-   null, `t`, `r`, `b`, `l`, `h`, `v`, ("all", sides, "horizontal", "vertical")
-   0..N multipliers for grid units (scale below), & `px` for 1 pixel

For example:

| Class         | CSS                                  |
| ------------- | ------------------------------------ |
| `bdl-u-p-3`   | `padding: 16px`                      |
| `bdl-u-ph-0`  | `padding-left: 0; padding-right: 0;` |
| `bdl-u-mr-px` | `margin-right: 1px;`                 |
| `bdl-u-mt-1`  | `margin-top: 4px`                    |

## Scale

Grid is based on 4px and uses scaling factors inspired by the [Tailwinds](https://tailwindcss.com/docs/customizing-spacing/) framework.

The `$bdl-grid-unit` constant is also available to be imported in SCSS and JS (as `bdlGridUnit`), for cases when you are not using utilities.
